# git commit
git 에 업로드를 할 때 사용하는 스킬입니다. 아래 3단계를 순차적으로 진행합니다.

## 1단계 : 초기화
`gh auth status` 로, github 로그인 되어있는지 확인합니다. 
이전단계에서 확인했다면, 이 단계는 건너뛰어도 됩니다.
로그인이 되어있지 않다면, 로그인을 시도하고 사용자에게 로그인을 요청합니다.

## 2단계 : 레포지토리 확인
연결된 레포지토리가 있는지 확인합니다. 
연결된 레포지토리가 없다면, 새로운 레포지토리를 만들거나 사용자에게 기존 레포지토리의 이름을 요청합니다.

## 3단계 : 문제 확인
git 에 업로드 하기 전, 충돌 문제가 없는지, 파일 오류 문제가 없을지 확인합니다.

## 4단계 : 업로드
git 에 commit 후 push 합니다. 
사용자가 커밋메시지를 지정하지 않으면, 알아서 입력해서 업로드 합니다.
