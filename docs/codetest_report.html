보안 점검 메모

개발용 CORS는 로컬 주소만 허용하고 있어 범위는 과도하지 않습니다.

SQL 접근은 확인된 범위에서 파라미터 바인딩 패턴을 사용하고 있습니다.

다만 인증/인가가 없어 내부 도구 수준을 넘는 배포에는 위험합니다.

파일 업로드 기능은 이번 대상 범위에 없어서 별도 취약점은 확인하지 못했습니다.

요약

FE 빌드와 BE 기동, DB health 응답은 정상이었습니다.

현재 가장 큰 리스크는 인증/인가가 전혀 없어 일정/팀원 변경 API가 그대로 노출된다는 점입니다.

SQLite는 외래키를 선언했지만 실제 강제가 빠져 있어 데이터 무결성 보완이 필요합니다.

미검증 항목

  • FE 단위 테스트 스크립트가 없어 자동 테스트는 빌드 위주로만 확인함
  • BE pytest 스위트가 없어 단위/통합 테스트는 별도 실행하지 못함
  • 브라우저 기반 UI 회귀 테스트는 수행하지 않음

권장 조치: 인증/인가 추가, SQLite 외래키 활성화, 테스트 스크립트 보강.
