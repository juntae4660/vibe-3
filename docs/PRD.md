# 공공직군 행정업무 슈퍼앱 PRD

## 1. 문서 목적

본 문서는 공공직군 행정업무 슈퍼앱의 초기 제품 요구사항을 정의한다. 개발 전 기능 범위, 사용자, 핵심 시나리오, 우선순위, 비기능 요구사항을 정리하여 FE/BE/DB 설계와 구현 기준으로 사용한다.

## 2. 제품 개요

공공직군 행정업무 슈퍼앱은 팀 단위 행정 실무자가 반복적으로 수행하는 일정 공유, 엑셀 정리, 민원 응대, 행정 뉴스 확인 업무를 하나의 웹 애플리케이션에서 처리하도록 돕는 업무 지원 플랫폼이다.

## 3. 목표

- 팀원의 휴가, 근무, 출장 일정을 한 화면에서 공유한다.
- 엑셀 파일을 기준 컬럼에 따라 자동 분리하거나 여러 파일을 병합한다.
- 민원 매뉴얼을 기반으로 대응 방식과 답변 스크립트 초안을 생성한다.
- 공공 행정 관련 뉴스 기사를 매일 아침 수집하고 요약 확인한다.
- 비개발자 행정 담당자도 별도 설치 없이 브라우저에서 사용할 수 있게 한다.

## 4. 주요 사용자

| 사용자 | 설명 | 주요 사용 기능 |
| --- | --- | --- |
| 팀장/관리자 | 팀 전체 근무 현황을 확인하고 업무 배분을 조정한다. | 스케줄 관리, 뉴스 확인 |
| 행정 실무자 | 엑셀 정리, 민원 응대, 자료 취합 업무를 수행한다. | 엑셀 자동화, 민원 챗봇 |
| 민원 담당자 | 민원 유형별 대응 기준과 답변 초안을 확인한다. | 민원 대응 챗봇 |
| 시스템 운영자 | 서비스 상태, 데이터 백업, 문서/매뉴얼 갱신을 관리한다. | 운영 관리, 데이터 관리 |

## 5. 기능 요구사항

### 5.1 팀원 스케줄 관리

#### 목적

팀원들의 휴가, 근무, 출장, 외근, 교육 등 일정을 공유하여 근무 공백과 업무 중복을 줄인다.

#### 주요 기능

- 월간/주간/일간 캘린더 조회
- 일정 유형 등록: 휴가, 근무, 출장, 외근, 교육, 기타
- 일정 생성, 수정, 삭제
- 팀원별 일정 필터링
- 일정 유형별 색상 구분
- 중복 일정 또는 장기 부재 일정 표시
- 오늘 일정 및 이번 주 주요 일정 요약

#### 입력 데이터

- 일정 제목
- 일정 유형
- 담당자
- 시작일시, 종료일시
- 장소
- 메모

#### 출력 데이터

- 캘린더 화면
- 팀원별 일정 목록
- 기간별 일정 요약

#### 우선순위

MVP 필수.

### 5.2 엑셀 업무 자동화

#### 목적

반복적인 엑셀 파일 분리/병합 작업을 자동화하여 행정 실무자의 수작업 시간을 줄인다.

#### 주요 기능

- 엑셀 파일 업로드
- 특정 컬럼 기준으로 파일 분리
- 여러 엑셀 파일 병합
- 컬럼명 자동 인식
- 처리 결과 미리보기
- 결과 파일 다운로드
- 처리 로그 확인

#### 세부 시나리오

- 사용자가 엑셀 파일을 업로드한다.
- 시스템이 시트와 컬럼 목록을 분석한다.
- 사용자가 기준 컬럼을 선택한다.
- 시스템이 기준 컬럼 값별로 파일을 분리한다.
- 사용자가 결과 ZIP 또는 엑셀 파일을 다운로드한다.

#### 제약사항

- 초기 버전은 `.xlsx` 형식을 우선 지원한다.
- 대용량 파일은 업로드 크기 제한과 처리 시간 제한을 둔다.
- 개인정보가 포함될 수 있으므로 업로드 파일은 처리 후 자동 삭제 정책을 적용한다.

#### 우선순위

MVP 필수.

### 5.3 민원 대응 챗봇

#### 목적

첨부된 민원 매뉴얼을 기반으로 민원 유형에 맞는 대응 방식, 확인 사항, 답변 스크립트 초안을 제공한다.

#### 주요 기능

- 민원 매뉴얼 업로드 및 관리
- 민원 내용 입력
- 관련 매뉴얼 조항 검색
- 대응 절차 제안
- 답변 스크립트 초안 생성
- 주의사항 및 추가 확인 항목 표시
- 답변 이력 저장

#### 챗봇 답변 원칙

- 매뉴얼 기반 답변을 우선한다.
- 근거가 부족한 경우 "확인 필요"로 표시한다.
- 법적 판단, 확정 처분, 민감 개인정보 처리에 대해서는 담당자 검토를 요구한다.
- 최종 답변은 사람이 검토 후 사용한다.

#### 입력 데이터

- 민원 내용
- 민원 유형
- 첨부 매뉴얼
- 부서 또는 업무 분야

#### 출력 데이터

- 대응 방향
- 확인해야 할 정보
- 답변 스크립트 초안
- 참고한 매뉴얼 항목

#### 우선순위

MVP 이후 고도화 대상. 단, 매뉴얼 업로드와 단순 검색 기반 답변은 초기 프로토타입에 포함 가능.

### 5.4 뉴스 기사 수집

#### 목적

공공 행정 관련 뉴스를 매일 아침 자동 수집하여 정책 변화, 사회 이슈, 행정 현안을 빠르게 파악한다.

#### 주요 기능

- 키워드 기반 뉴스 수집
- 매일 아침 정기 실행
- 기사 제목, 출처, 발행일, 링크 저장
- 중복 기사 제거
- 키워드별 분류
- 주요 기사 목록 조회
- 기사 요약 또는 메모 기능

#### 기본 키워드 예시

- 공공행정
- 지방자치
- 민원
- 복지행정
- 행정안전부
- 공무원
- 정책

#### 제약사항

- 뉴스 수집은 각 사이트의 이용약관과 robots 정책을 준수해야 한다.
- 초기 버전은 공개 RSS 또는 공식 검색 API 사용을 우선한다.
- 저작권 문제를 피하기 위해 기사 전문 저장은 지양하고 제목, 링크, 짧은 요약 중심으로 저장한다.

#### 우선순위

MVP 이후 2차 기능.

## 6. MVP 범위

MVP에 포함할 기능은 다음과 같다.

- 팀원 스케줄 캘린더 조회/등록/수정/삭제
- 엑셀 파일 업로드
- 기준 컬럼별 엑셀 분리
- 여러 엑셀 파일 병합
- FastAPI 기반 API 서버
- SQLite 기반 로컬 데이터 저장
- 기본 운영 문서와 실행 방법

MVP에서 제외하거나 후순위로 둘 기능은 다음과 같다.

- 챗봇 LLM 연동
- 뉴스 정기 수집 자동 스케줄러
- 권한별 세부 접근 제어
- 외부 메신저/메일 알림
- 감사 로그 대시보드

## 7. 비기능 요구사항

### 7.1 보안

- 개인정보 포함 가능성이 있는 파일은 서버에 영구 저장하지 않는다.
- 업로드 파일은 처리 완료 후 삭제하거나 보관 기간을 명시한다.
- 민원 데이터와 답변 이력은 접근 권한을 제한한다.
- 추후 운영 환경에서는 HTTPS 적용을 전제로 한다.

### 7.2 성능

- 일반 일정 조회는 1초 이내 응답을 목표로 한다.
- 10MB 이하 엑셀 파일은 10초 이내 처리 완료를 목표로 한다.
- 대용량 파일은 비동기 처리 또는 처리 제한 안내를 제공한다.

### 7.3 사용성

- 행정 실무자가 별도 교육 없이 사용할 수 있는 단순한 화면 구조를 지향한다.
- 파일 업로드, 처리 상태, 다운로드 결과를 명확히 표시한다.
- 오류 발생 시 사용자가 조치 가능한 메시지를 제공한다.

### 7.4 확장성

- 기능별 모듈을 분리하여 일정, 엑셀, 챗봇, 뉴스 기능을 독립적으로 확장할 수 있게 한다.
- SQLite로 시작하되 향후 PostgreSQL 등 서버형 DB로 이전 가능한 구조를 고려한다.
